19//! Plugin driver.
20//!
21//! Driver is the top-level object of AudioServer plugin. It contains Plugin
22//! object, which is the root of the whole object hierarchy. The driver's job
23//! is to receive requests from HAL and redirect them to appropriate objects.
24//!
25//! Driver is exposed to HAL as "driver reference" (@c AudioServerPlugInDriverRef).
26//! The driver reference is actually a pointer to pointer to "plugin driver interface"
27//! (@c AudioServerPlugInDriverInterface), which is a struct with a bunch of function
28//! pointers for various operations (mostly for property dispatch and I/O).
29//!
30//! Driver fills this struct with its private functions. Each of them just finds
31//! the registered object by ID and redirects operation to it. Objects are searched
32//! in Context::Dispatcher. Every object requires Context as constructor argument
33//! and automatically registers and unregisters itself in Dispatcher.
34//!
35//! By default driver creates its own Context and Plugin objects, but you can
36//! provide your own if desired.
37//!
38//! After constructing driver, you need to use GetReference() method to obtain
39//! the "driver reference" and return it from the plugin entry point, like:
40//!
41//! @code
42//! extern "C" void* MyEntryPoint(CFAllocatorRef, CFUUIDRef typeUUID)
43//! {
44//! if (!CFEqual(typeUUID, kAudioServerPlugInTypeUUID)) {
45//! return nullptr;
46//! }
47//! static auto driver = std::make_shared<aspl::Driver>();
48//! return driver->GetReference();
49//! }
50//! @endcode
51//!
52//! Don't forget do declare your entry point in Info.plist of the plugin.
53class Driver
54{
55public:
